# Energy Grid Game — 교육 강화 로드맵

> **목표**: Townscaper 스타일 빌더 위에 **실제 한전·KPX 현장 운영 체계**를 게임 메커니즘으로 녹여,
> 플레이어가 자연스럽게 전력산업 실무를 학습하는 콘텐츠로 발전시킨다.

---

## 📐 전체 구조

```
┌──────────────────────────────────────────────────────┐
│  Phase 1 — 운영 기능 (Mechanics)                     │
│   계통제약 → 보호계전 → DR → 블랙스타트              │
│   (게임에서 "무슨 일이 일어나는가")                   │
├──────────────────────────────────────────────────────┤
│  Phase 2 — 교육 레이어 (Pedagogy)                    │
│   L1 호버카드 / L2 사후리포트 / L3 위키 /            │
│   L4 코칭 / L5 시나리오 / L6 진척도                  │
│   (게임이 "어떻게 가르치는가")                       │
├──────────────────────────────────────────────────────┤
│  통합 — 단계적 해금 (Progression)                    │
│   점수·경험에 따라 기능·콘텐츠가 순차 해금           │
└──────────────────────────────────────────────────────┘
```

## 🔧 Phase 1 — 4대 운영 기능

### 1-A. 계통제약 (Grid Constraint)
**핵심**: 송전선 용량 한계와 부하율 시각화

- `TILE_TYPES`에 `lineCapacity` 추가 (송전 200MW / 배전 50MW)
- `simulate()` 후처리: edge별 flow 추정 (단일경로=downstream demand 합 / 병렬=균등 분배)
- return에 `flows: Map<edgeKey, {load, capacity, ratio, overload}>`
- `PowerLine`에 `loadRatio` prop → 황금→노랑→빨강 색상 lerp + overload 시 깜박임
- HUD에 "혼잡선로 N개" 표시

**난이도**: ★★☆☆☆ (3~4h)

### 1-B. 보호계전 (Protective Relay)
**핵심**: 자동 트립 + 재폐로 시퀀스

- 새 상태: `breakerStates: Map<edgeKey, {state, tripTime}>`
- 자동 트립 트리거:
  - 계통제약(1-A) overload 5초 지속
  - 번개 직격 인접 선로
- 재폐로 시퀀스 (실제 한전 동작 모사):
  - 트립 → 0.5s → 1차 재폐로
  - 실패 → 2s → 2차 재폐로
  - 실패 → lock out (수동 복구 필요)
- 차단기 시각 표시 (송전탑/변전소 위 작은 사각형, 녹/적/회색)
- 트립 사유 토스트: "선로 용량 초과", "낙뢰 검출", "단락 의심"

**난이도**: ★★★☆☆ (4~5h, 1-A 의존)

### 1-C. 수요관리 (DR — Demand Response)
**핵심**: 피크 시간 수요 감축, 인센티브 보너스

- `simulate(buildings, disabledKeys, disabledEdges, drActive)` — drActive 건물 demand × 0.5
- "DR 발령" 버튼 (수요배율 > 1.2 또는 freq < 59.7 일 때만 활성)
- 발령: factory 중 자동 선정 → 30s 감축 → 60s 쿨다운
- 이벤트형: "산업체 DR 신청" 가끔 발생 → 수락/거절
- DR로 freq 안정 유지 시 점수 1.5배 보너스

**난이도**: ★★☆☆☆ (3h, 독립)

### 1-D. 블랙스타트 (Black Start)
**핵심**: 전계통 정전 후 순차 복구

- 새 `TILE_TYPE`: `hydroPlant` (블랙스타트 가능 발전소)
- 트리거: |freq - 60| > 2.5Hz가 5초 이상 → 캐스케이딩 트립 연출 → 블랙스타트 모드
- 상태머신:
  ```
  idle → blackout (전부 disabled, 어두워짐)
       → bs_phase1: 수력 1기 기동
       → bs_phase2: 송전선 1개씩 가압
       → bs_phase3: 변전소 도달 → 작은 부하 픽업
       → bs_phase4: 다음 발전기 동기화 (sin파 정렬 미니게임)
       → bs_phase5: 점진적 부하 추가
       → recovered
  ```
- 동기화 미니게임: 두 sin파(기존 60Hz vs 신규) 위상 정렬 → "투입" 버튼
- 페널티: 너무 빨리 부하 추가 → 재트립 → phase2부터

**난이도**: ★★★★★ (2~3일, 별도 컴포넌트)

## 📚 Phase 2 — 6대 교육 레이어

### L1. 호버 학습 카드 (★ 최우선)
- 건물/선로/이벤트마커/HUD 호버 시 맥락 카드
- 정보 구성: 역할 + 실제 값(전압·용량) + 현재 상태 + 학습 한 줄
- 게임 흐름 방해 없음

**난이도**: ★☆☆☆☆

### L2. 사건 사후 리포트
- 트립/블랙아웃/DR 종료 시 닫을 수 있는 패널
- 구성: 시각 + 위치 + 원인 + 동작 + 실제 현장 비교
- [자세히] 누르면 L3 위키 해당 페이지로

**난이도**: ★★☆☆☆

### L3. 용어 사전 / 위키 (사이드패널)
- 햄버거 메뉴 또는 우측 슬라이드 패널
- 카테고리: 발전 / 송전 / 배전 / 운영 / 비상
- 항목별: 정의 → 게임 매핑 → 실제 현장 → 관련 항목
- ★ 표시로 "게임에서 경험한" 항목 강조

**난이도**: ★★★☆☆ (콘텐츠 작성 비중 큼)

### L4. 실시간 코칭
- 좌측 하단 페이드 인/아웃 힌트
- 트리거 예:
  - freq 경고 zone > 3s → "공급 부족. 발전소 증설 또는 DR 발령 고려"
  - 선로 부하 > 90% × 5s → "이 선로 곧 트립. 우회 경로 확보"
  - 변전소 없이 가정만 → "변전소 없이는 저압망 통전 불가"
  - 단일 발전소 의존 → "N-1 위반 위험"
- 동일 힌트 1분 쿨다운

**난이도**: ★★☆☆☆

### L5. 학습 시나리오 (미니 캠페인)
- "오늘의 챌린지" 형식, 짧은 미션
- 시나리오 목록:
  - **휴전작업 절차**: 9단계 체험 (작업계획→우회→검전→접지→작업→송전)
  - **여름철 피크**: 수요 150% DR로 살아남기
  - **N-1 분석**: 가장 치명적인 선로 찾기
  - **블랙스타트**: 전계통 정전 복구
  - **까마귀 다발**: 동시 다발 우선순위 판단
- 별도 모드로 진입 (App.jsx에 메뉴 추가)

**난이도**: ★★★★☆

### L6. 학습 진척도
- 사이드 패널에 진척도 표시
- 상태 단계: 🔒 잠김 → 🔓 해금 → ✅ 경험 → ⭐ 숙달
- 숙달 기준 예:
  - 트립 후 30초 내 복구 → 보호계전 숙달
  - DR 발령으로 freq 회복 → DR 숙달
- LocalStorage 저장

**난이도**: ★★★☆☆

## 🔓 통합 — 단계적 해금

플레이어가 한꺼번에 다 받지 않도록 점수·경험으로 순차 공개.

| 누적 MWh | 해금 |
|---|---|
| 0 | 기본 건설 + 주파수 유지 (현재) |
| 6 | 영역 r5 + L1 호버카드 활성화 |
| 18 | 영역 r6 + L4 실시간 코칭 |
| 30 | **1-A 계통제약** (선로 부하 시각화) |
| 40 | 영역 r7 + L2 사후 리포트 |
| 60 | **1-B 보호계전** (트립/재폐로) |
| 75 | 영역 r8 + L3 위키 |
| 100 | **1-C DR 발령 버튼** + L6 진척도 |
| 200 | **1-D 블랙스타트 챌린지** + L5 시나리오 모드 |

## 🗓️ 작업 순서 (의존성 최적화)

```
Week 1
├─ L1 호버카드           [기반: 학습 정보를 어떻게 보여줄지 패턴 확립]
└─ 1-A 계통제약          [선로 부하 데이터 → L1 카드에 즉시 활용]

Week 2
├─ L4 실시간 코칭        [데이터는 모두 있음, UX 추가]
└─ 1-B 보호계전          [1-A 위에 자연스럽게]

Week 3
├─ L2 사후 리포트        [1-B 트립 사건에 가장 잘 어울림]
└─ 1-C DR                [독립 기능, 사이에 환기용]

Week 4
├─ L3 위키 (콘텐츠 작성) [시간이 가장 많이 듦, 평행 진행]
└─ L6 진척도            [누적된 경험 기록 시스템]

Week 5~6
└─ 1-D 블랙스타트        [위 모두를 활용하는 엔드게임]

Week 7~8
└─ L5 학습 시나리오 5종  [기존 메커니즘 위에 시나리오 미션]
```
